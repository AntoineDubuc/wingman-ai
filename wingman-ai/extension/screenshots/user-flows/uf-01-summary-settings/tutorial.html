UF-01: Configure Summary Settings

Wingman AI -- Options page summary configuration walkthrough

1
Open Options Page

Navigate to the Wingman Settings page. The full options page is displayed with all configuration sections: API Keys, Speaker Filter, Google Drive Integration, Call Summary, Knowledge Base, and System Prompt.

Verify: The page loads with the orange Wingman header and all settings cards are visible.
2
Scroll to Summary Card

Scroll down to the "Call Summary" settings card. Both toggles -- "Auto-generate call summary" and "Include key moments" -- are enabled (orange) by default.

Verify: Both "Auto-generate call summary" and "Include key moments" toggles appear in the ON (orange) position.
3
Disable Auto-Summary

Click the "Auto-generate call summary" toggle to disable it. The toggle switches to the OFF (gray) position.

Verify: The "Auto-generate call summary" toggle is now gray/OFF. The "Include key moments" row becomes visually grayed out.
4
Verify Moments Toggle Grays Out

With auto-summary disabled, the "Include key moments" toggle is grayed out and non-interactive. This is a dependent setting -- key moments cannot be configured without auto-summary enabled.

Verify: The "Include key moments" label and description text appear faded. The toggle shows a muted/disabled state.
5
Re-enable Auto-Summary

Click the "Auto-generate call summary" toggle again to re-enable it. Both toggles return to their active (orange) state.

Verify: Both toggles are back to ON (orange). The "Include key moments" text is no longer faded.
6
Disable Key Moments Only

With auto-summary still enabled, click the "Include key moments" toggle to disable it independently. The auto-summary toggle remains ON while key moments is OFF.

Verify: "Auto-generate call summary" is ON (orange). "Include key moments" is OFF (gray). Both labels remain at full opacity since the parent toggle is enabled.
7
Settings Persisted After Reload

Reload the options page to confirm settings are persisted via chrome.storage. Both toggles should reflect the saved state from the previous step.

Verify: After reload, the toggles match the last saved configuration. Settings survive page navigation and browser restarts.
